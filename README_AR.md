# 📊 نظام حفظ بيانات المستشعرات

## المميزات

✅ حفظ قراءات المستشعرات تلقائياً كل **دقيقة واحدة**  
✅ قاعدة بيانات SQLite محلية  
✅ تخزين جميع البيانات (درجة الحرارة، الرطوبة، الغاز، الضوء، التربة)  
✅ عرض البيانات المحفوظة بشكل منظم

---

## البيانات المحفوظة

| الحقل | الوصف |
|------|-------|
| **timestamp** | وقت القراءة |
| **temp** | درجة الحرارة (°C) |
| **humidity** | الرطوبة (%) |
| **gas_raw** | قيمة الغاز الخام |
| **gas_alert** | تنبيه الغاز (نعم/لا) |
| **light_pct** | كثافة الضوء (%) |
| **light_raw** | قيمة الضوء الخام |
| **soil_pct** | رطوبة التربة (%) |
| **soil_raw** | قيمة التربة الخام |

---

## طريقة الاستخدام

### 1. تثبيت المكتبات
```bash
pip install -r requirements.txt
```

### 2. تشغيل برنامج قراءة المستشعرات
```bash
python esp32_client.py
```
سيعرض القراءات مباشرة ويحفظها كل دقيقة تلقائياً

### 3. عرض البيانات المحفوظة
```bash
python view_data.py
```
اختر من الخيارات:
- آخر 20 قراءة
- قراءات اليوم
- آخر 24 ساعة
- آخر 7 أيام

---

## الملفات

- **esp32_client.py** - برنامج قراءة البيانات من ESP32 وحفظها
- **database.py** - نموذج SQLAlchemy وعمليات قاعدة البيانات
- **view_data.py** - برنامج عرض البيانات المحفوظة
- **requirements.txt** - المكتبات المطلوبة
- **sensor_data.db** - قاعدة البيانات (ستُنشأ تلقائياً)

---

## نصائح

💡 يمكنك تغيير `INTERVAL` في `esp32_client.py` لتغيير عدد مرات القراءة من ESP32  
💡 يمكنك تغيير `SAVE_INTERVAL` لتغيير عدد مرات حفظ البيانات في قاعدة البيانات  
💡 قاعدة البيانات `sensor_data.db` تُخزن في نفس المجلد

---

## أمثلة على استخدام قاعدة البيانات برمجياً

```python
from database import save_sensor_reading, get_latest_readings

# حفظ قراءة
data = {
    "temp": 25.5,
    "humidity": 60.0,
    "gas_raw": 400,
    "gas_alert": False,
    "light_pct": 75,
    "light_raw": 800,
    "soil_pct": 50,
    "soil_raw": 500
}
reading = save_sensor_reading(data)

# الحصول على آخر 10 قراءات
readings = get_latest_readings(10)
for reading in readings:
    print(reading)
```
